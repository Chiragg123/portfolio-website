Projects

Production Android Apps

Industrial-grade applications built with Java & Kotlin, integrating REST APIs, hardware sensors, and complex business logic.

PROJECT 01 ↗
Industrial Packing & QC App

A comprehensive quality control application for industrial packing lines. Validates cases via barcode, loads inspection data from APIs, and persists QC outcomes — all in a dynamic, real-time interface.

  • Barcode scanning for case validation & inspection trigger
  • API-based inspection data loading (/api/IsCaseRework/v1)
  • Dual dynamic RecyclerViews for inspection & BOM items
  • QC pass/fail data saved via API (/api/QCPass/v1)
  • Real-time form validation & business logic enforcement
Java Android REST API RecyclerView Barcode
PROJECT 02 ↗
Secondary Packing with Weight Validation

An Android app that interfaces directly with industrial weight machines over IP/Port, reads live weight data, validates against configurable min/max thresholds, and auto-saves when conditions are met.

  • Live weight reading from machine via IP+Port socket
  • Min/Max weight validation with configurable thresholds
  • Auto-save trigger when all valid conditions are satisfied
  • BroadcastReceiver for real-time data streaming updates
  • Visual feedback & alert system for threshold violations
Kotlin Android Hardware I/O BroadcastReceiver Socket
PROJECT 03 ↗
Container Loading System

A dashboard-driven app for managing container loading plans in real time. Operators scan pallets to trigger API calls, while quantities update dynamically per product selections.

  • Dashboard with loading plans & progress visualization
  • Pallet scan API integration (/api/v1/ScanPallet)
  • Dynamic quantity counters updating in real-time
  • Spinner-based product & container selection
  • Session-based workflow management
Java Android REST API Barcode Dashboard
PROJECT 04 ↗
Rework Management System

An Android application for managing QC-failed items through a structured rework workflow. Fetches fail items from the API, presents dropdown action options, and persists rework decisions back to the server.

  • Fetch & display QC-failed items from API endpoint
  • Dropdown-driven rework action selection per item
  • Dynamic UI rendering based on API response schema
  • Save rework actions and status via API
  • Status tracking & confirmation workflows
Kotlin Android REST API RecyclerView Dynamic UI
